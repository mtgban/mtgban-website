‚å®Ô∏è Basic Syntax

You can search using the Pricefall bot notation:

name[|code[|number]]{&*~`}

Append a special character to filter by finish:

SuffixMeaning
&Non-foil only
*Foil only
~Etched only
`Alt-foil (surge, ripple, galaxy, etc)

Note: This syntax is disabled if you use a regular expression option to avoid aliasing.

There is also limited support for human readable tags. For example, appending (Extended Art) to a card name will show only Extended Art versions. This won't work with regexp search mode.

You can set options in any order and any amount. Use comma , to separate multiple values.

Lightning Bolt|LEA
Lightning Bolt from Alpha
Sol Ring*
Foil Sol Rings only
Sheoldred (Showcase)
Showcase versions of Sheoldred
r:rare,mythic
Rare OR mythic cards

üìö Editions and Sets

Filter cards by edition using the Scryfall notation s:CODE and the full edition name (enclosed in quotes) s:"Aether Revolt".

Regular expressions are supported with the se: option.

As a compatibility layer, e:CODE and ee:CODE are supported for their respective cases.

s:MKM
Cards from Murders at Karlov Manor
s:"Aether Revolt"
Using full set name
se:^MH
Sets starting with "MH" (regex)
e:LEA
Compatibility syntax for Alpha

üî¢ Collector Numbers

Filter by collector number using the Scryfall notation cn:NUMBER.

For simple numbers (without prefix or suffix) you can use cn>NUMBER and cn<NUMBER, or a range: cn:NUMBER-NUMBER.

Regular expressions are supported with cne:REGEXP.

Prepend the set code like cn:CODE:NUMBER to filter only cards from that set, leaving other results untouched.

cn:123
Cards numbered 123
cn:1-50
Cards numbered 1-50
cn>300
Cards above #300
cn:MKM:42
#42 from MKM only

üí∞ Prices and Ratios

Filter by price with price>VALUE or price<VALUE to exclude results outside your range.

Filter by buylist price ratio with ratio>VALUE. Higher ratios indicate vendors are paying more relative to retail.

price>10
Cards above $10
price<5
Cards under $5
ratio>80
High buylist demand (80%+)

üè™ Stores and Vendors

Filter by seller/vendor name with store:shorthand, or specific types with seller:shorthand and vendor:shorthand.

These filters will drop any result where the specified scraper is not present. If you want to see results from a specific scraper only, prefix with only: store:only:shorthand.

Filter results to include only retail or buylist prices with skip:retail and skip:buylist.

Skip results that do not have retail or buylist prices with skip:emptyretail and skip:emptybuylist, or results without any prices at all with skip:empty.

Filter by store region with region:VALUE using us, eu, or jp.

Note: These options leave index results available. To hide them, use skip:index (or more specifically skip:indexretail and skip:indexbuylist).

store:TCG
TCGplayer listings
store:only:CK
Only Card Kingdom results
vendor:CK
Card Kingdom buylist
region:eu
European stores only
skip:index
Hide index/aggregate prices

üîé Search Modes

Change search behavior with sm:VALUE:

ModeBehavior
exactExact name match (default)
prefixNames starting with search term
anyNames containing search term
regexpRegular expression (case sensitive)
scryfallForward search to Scryfall

In scryfall mode, BAN card filters will be disabled to avoid conflicting results between the two search syntaxes, while store and price filters will still work.

Vesuva

sm:prefix "Dragonlord"
prefix: Cards starting with "Dragonlord"
sm:any *Draco*
any: Draco and all cards with "draco"
sm:regexp Cluestone$
regexp: Cards ending in "Cluestone"
sm:scryfall art:loot f:f
scryfall: Foil cards tagged "loot"

üí° General Tips

Historical Data: Access price history from a few major vendors by clicking üìä for each card.

Data Refresh: Data is refreshed periodically over the day.

Entry Format: Entries are formatted as card name (finish) - edition (collector #) - # of prints.

Source Products: Retrieve the source product of a card or sealed product by pressing "Found in * products". Similarly, pressing üìñ on a card will return every possible product containing any kind of reprint of that card.

Price Ratio: The percentage found in buylist offers is the price ratio ‚Äî the higher it is, the more a vendor is looking to purchase that card. This percentage is available only if the seller is selling copies of the same card at the same conditions.

Conditions: Inventory prices refer to the stated conditions (and their accuracy depends on the data provider). Buylist prices are always referring to NM conditions.

Flavor Names: When searching for the flavor name of a card, only those specific versions will appear (unless you disable this option in the Other Preferences section). However this only works when searching for a single specific card, and won't be available for complex search queries (they will still be performed using the canonical name of the card).

TCG Low: TCG Low is a special value referring to TCG algorithms that may differ from quantity and quality of listings.

Trade-In: If you hover on a buylist price, a small tooltip will appear displaying the corresponding trade-in price, if available.

Feedback: In case of mistakes or incongruities, please notify the devs in the BAN Discord.

Support BAN: Should you find this content useful, consider clicking on one of the provided links to make a purchase on the website, and directly support BAN.
